# Session Diary Guidance

Use session diaries for **recurring threads or operating contexts** (e.g. bootstrap adoption, release train, long feature arc).

## File pattern

`session/<thread-slug>.md` — one file per recurring context.

## Entry format (compact)

```markdown
## YYYY-MM-DD
- Context:
- Decisions:
- Open loops:
- Follow-ups:
- Norms learned (thread-specific):
```

## Rules

- Capture substance needed to resume, not verbatim chat.
- Link to specs, issues, PRs, and backlog IDs.
- Close or archive threads when the arc completes; promote durable facts upward.

## Promotion

When a diary entry becomes project truth, copy summarized facts to `project/CONTINUITY.md` and trim the diary to pointers.
